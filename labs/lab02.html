LAB 2

Intro

In this lab, you will practice basic HTML and CSS by setting up your initial webpages. This lab relies on Lectures from Week 2. You can revisit lectures’ recordings from Yuja or find slide decks in Modules. 

The lab will have three parts: a follow-along step-by-step task, a creative task with less cues, and a self-narration.

 

Work in Pairs!

You will work in pairs for this lab. For general details on this, revisit Lab 1.

For this lab specifically

  1. The follow-along task should be done individually! Consult with your partner if something is not working. Help your partner if they are experiencing troubles.
  2. The creative task could be done in two ways.
    1. Individually, while helping each other, as the previous task. Each partner does and submit their own results
    2. In close collaboration involving pair programming. In this case, you can have shared results. Each of you should have a copy of those results in their ART101 folder. You will need to narrate in a compelling way how your collaboration worked, what were the obstacles and findings. Since there were two people working on the same code, we will expect more details and experimentations and will grade accordingly to those expectations.
  3. In any case, each partner submits a PDF to Canvas and makes a self-eval on Canvas beyond the narration.

Task 1. Follow Along: Build Your Homepage

In this task, you will create the first homepage for your ART101 website.

Keep testing your page in Chrome after each major step. This will help you notice exactly what changed, and when something went wrong.

Do not forget to save your files after every edit.

1. Open your project in Visual Studio Code

Open Visual Studio Code.

If your ART101 folder is not already open:

  1. Go to File > Open Folder
  2. Find your ART101 folder
  3. Open it

What is happening here: you are opening your main website project so all files are visible and editable in one place.

2. Check what's inside

In the VS Code file panel, make sure your project looks roughly like this based on our class activities:

What is happening here: you are organizing your site before adding content. Good structure makes links and file paths easier to manage.

3. Open or create index.html

In the main ART101 folder:

  • open index.html if it already exists
  • or create a new file named index.html if it does not exist yet

If the file contains old practice code, delete it so you can start fresh.

What is happening here: index.html is the main homepage file of your website. It is usually the first page a browser looks for.

4. Add the starter HTML template

Copy and paste this code into index.html:

<!doctype html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <title>Ilia's ART101 Homepage</title>
    <link href="site.css" rel="stylesheet" type="text/css">
</head>
<body>

    <!-- All visible page content goes between <body> and </body> -->

</body>
</html>

Then save the file.

What is happening here:

  • <!doctype html> tells the browser this is an HTML5 document
  • <html> wraps the whole page
  • <head> stores page information for the browser
  • <title> sets the text shown on the browser tab (change it it your own!)
  • <link> connects your HTML file to site.css we will create later
  • <body> contains everything visible on the page

5. Add a main heading with <h1>

Inside the <body> ... </body> area, add:

<h1>This is Ilia's ART101 website homepage</h1>

Change the text to your own version.

For example:

<h1>Maya's brilliant website homepage</h1>

What is happening here: <h1> creates the main heading of the page. It is usually the biggest and most important title.

6. Add a subheading with <h2>

Below your <h1>, add:

<h2>Welcome folks!</h2>

What is happening here: <h2> creates a second-level heading and starts a new section of the page.

7. Add a paragraph about yourself

Below the <h2>, add a paragraph:

<p>
    Several sentences about me... You can use your Discord intro or write something new.
</p>

Replace the placeholder text with your own writing.

What is happening here: <p> creates a paragraph of regular text.

8. Add another subheading and an image

Add another section heading:

<h2>This is totally me:</h2>

Then add an image below it:

<img src="mememe.png">

Now complete these steps carefully:

  1. Choose an image of yourself, or any image you want to use.
  2. Copy that image file into your main ART101 folder. Or download it to ART101 folder if it's online.
  3. Rename the image if needed so the file name is all lowercase and has no spaces.
  4. Replace mememe.png in the src="..." attribute with your actual file name.

For example, if your image file is named myphoto.jpg, write:

<img src="myphoto.jpg">

What is happening here: the <img> tag places an image on the page. The src attribute tells the browser which file to load.

Common problem: if the image does not appear, the file name in src="..." probably does not exactly match the real file name, or the image file does not sit directly in ART101 folder.

9. Add another subheading and an ordered list

Add a new heading:

<h2>These are my exercises and environs:</h2>

Then add this list below it:

<ol>
    <li>Links to my follow-along exercises</li>
    <li><a href="/exercises/ex01.html">Exercise 1</a></li>
    <li>Links to iterations of my creative sandbox</li>
    <li><a href="/environ/en01.html">Environ 1</a></li>
</ol>

What is happening here:

  • <ol> creates an ordered list
  • <li> creates each list item
  • <a> creates a link
  • href tells the link where to go

You can change the text of the list items to match your own project.

10. Review your full index.html

At this point, your file should look something like this:

<!doctype html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <title>Ilia's ART101 Homepage</title>
    <link href="site.css" rel="stylesheet" type="text/css">
</head>
<body>

    <h1>This is Ilia's ART101 website homepage</h1>
    <h2>Welcome!</h2>

    <p>
        Several sentences about me... You can use your Discord intro or write something new.
    </p>

    <h2>This is totally me</h2>
    <img src="mememe.png">

    <h2>These are my exercises and environs:</h2>
    <ol>
        <li>Links to my follow-along exercises</li>
        <li>Links to iterations of my creative sandbox</li>
        <li><a href="/environ/en01.html">Environ 1</a></li>
    </ol>

</body>
</html>

Save the file.

Then open index.html in Chrome and check that:

  • text appears
  • headings appear at different sizes
  • the image loads
  • the list appears
  • the link looks clickable

What is happening here: you are testing your HTML before adding CSS. This helps you separate content problems from styling problems.

11. Create and open site.css

In the main ART101 folder, create a file named site.css if it does not already exist.

Open it in VS Code.

What is happening here: site.css is your stylesheet already linked from index.html head section. It controls how the HTML page looks.

12. Style the <body>

Add this code to site.css:

body {
    background: #e1f3f1;
    padding: 50px;
}

Save the file, then refresh Chrome.

What is happening here:

  • body targets the whole visible page
  • background changes the background color
  • padding adds inner space around the content

You are encouraged to modify the values and experiment.

13. Style the headings

Below the previous code, add:

h1,
h2,
h3 {
    color: #1d1184;
    font-family: sans-serif;
}

Save the file, then refresh Chrome again.

What is happening here:

  • h1, h2, h3 means these styles apply to all three heading types
  • color changes the text color
  • font-family changes the typeface

14. Style the image

Below that, add:

img {
    width: 200px;
}

Save and refresh Chrome.

What is happening here: this CSS rule makes all images on the page display at 200px wide.

15. Final check

Your page should now have:

  • a background color
  • space around the content
  • styled headings
  • a smaller image
  • organized sections

Something like this:

If something looks wrong, check these first:

  • Did you save both index.html and site.css?
  • Is site.css spelled correctly in both the file name and the <link> tag?
  • Is the image file name correct?
  • Is all page content inside <body> ... </body>?
  • Did you refresh Chrome?

  2. Take a screenshot of your VS Code workspace with site.css open and your code visible. Add this screenshot to your PDF.
  3. Take a screenshot of your final page in Chrome. Add this screenshot to your PDF.

Task 2. Initiating the Environment

This task begins a longer series of creative programming experiments.

Unlike the previous follow-along task, this one gives you general cues instead of detailed instructions. The goal is to give you room to experiment, play, break things, fix them, and discover new tricks along the way. That is a real part of programming and creative work.

Check the rubric: for this task, you will not be heavily penalized if something does not work perfectly or behaves strangely. The main point is to show self-guidance, experimentation, and project thinking.

Keep testing your page in Chrome as you work. Even if the page is incomplete, checking it often will help you notice what is changing.

Do not forget to save your files after every edit.

1. Imagine your environment

Think about what kind of environment you want to develop in this series of tasks throughout the course.

Your environment could be:

  • a rainforest
  • a Pokémon universe
  • yet another Star Wars spin-off
  • the UCSC campus
  • an abstract crystal dimension
  • your inner world
  • a horror story
  • or something else entirely

Brainstorm. Make doodles. Write a few notes. Or just choose the first strong idea that comes into your head.

The main point: this environment should feel interesting to you. It should make you curious, engaged, and excited to keep building it over time.

What is happening here: you are choosing the world that will guide your design choices, mood, visuals, and future experiments.

2. Create env01.html

Inside the environ/ folder of your main ART101 project, create a new file named env01.html.

Your folder should now look something like this:

ART101/
    index.html
    site.css
    exercises/
    environ/
        env01.html

What is happening here: env01.html will become the first webpage in your environment series.

3. Create env.css

Inside the same environ/ folder, create another file named env.css.

Your folder should now look something like this:

ART101/
    index.html
    site.css
    exercises/
    environ/
        env01.html
        env.css

What is happening here: env.css will control the visual style of your environment page.

4. Connect the HTML and CSS files

Use env01.html for your HTML content and env.css for your CSS styles.

Your env01.html file should include a basic HTML structure and a link to env.css.

For example:

<!doctype html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <title>Environment 01</title>
    <link href="env.css" rel="stylesheet" type="text/css">
</head>
<body>

</body>
</html>

What is happening here: the HTML file builds the page structure, and the CSS file controls how that structure looks.

5. Add a background that sets the mood

Create a background that establishes the feeling of your environment.

This background can be:

  • a color
  • an image
  • or a combination of both

Try applying the background to the body tag so the entire page feels immersive.

For example:

  • a deep green color for a rainforest
  • a star field image for a space setting
  • a dark red or black background for a horror scene

What is happening here: the background creates the first emotional impression of your page.

6. Add a paragraph that introduces the environment

Add a paragraph of text that introduces visitors to this world.

Make it feel narrative, atmospheric, or dramatic.

For example, you could write something in the spirit of:

A long time ago in a galaxy far, far away...

But write your own version for your own environment.

What is happening here: this paragraph gives the viewer a sense of place, tone, and story.

7. Add the first character or object as an image

Add your first character, creature, object, or element of the world using the <img> HTML tag.

This image might be:

  • a person
  • a monster
  • a tree
  • a spaceship
  • a building
  • an abstract object

Place the image file in a location where your HTML file can find it, then connect it using the src attribute.

For example:

<img src="creature.png">

Important: make sure the file name matches exactly, including the file extension such as .jpg, .png, or .webp.

What is happening here: you are introducing the first visible element inside your environment.

8. Put the pieces together

Your page should include at least these three ingredients:

  • a background that sets the mood
  • a paragraph that introduces the environment
  • an image of the first character or object

You may add more if you want, but these three parts are the minimum starting point.

What is happening here: you are building the first version of an original world that you will keep developing later.

9. Test your page in Chrome

Open env01.html in Chrome and check:

  • Does the background appear?
  • Does the paragraph show up?
  • Does the image load correctly?
  • Does the page create a clear mood or atmosphere?

If something is missing, check:

  • Did you save both env01.html and env.css?
  • Did you link env.css correctly in the <head>?
  • Is the image file name correct?
  • Is the image file in the correct folder?

WHAT TO SUBMIT

  1. Take a screenshot of your VS Code workspace with env01.html open and your code visible. Add this screenshot to your PDF.
  2. Take a screenshot of your VS Code workspace with env.css open and your code visible. Add this screenshot to your PDF.
  3. Take a screenshot of your resulting page in Chrome. Add this screenshot to your PDF.

This task is supposed to be more open. You are making choices on your own, and that is part of the assignment.

There is no single correct result. Your environment can be strange, poetic, messy, funny, eerie, minimal, dramatic, cute, broken, or unexpected.
